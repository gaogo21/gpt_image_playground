# 如何找到"视频生成 DS 2.0"配置 - 详细图文

## 问题：看不到"视频生成 DS 2.0"配置？

**原因**：浏览器使用了旧的缓存数据

**解决方案**：清除缓存或强制刷新

---

## 🔄 方法 1：强制刷新页面（推荐）

### Windows / Linux
按 **Ctrl + Shift + R** 或 **Ctrl + F5**

### Mac
按 **Cmd + Shift + R**

然后重新打开设置，就能看到"视频生成 DS 2.0"了。

---

## 🗑️ 方法 2：清除应用数据（彻底解决）

### 步骤 1：打开开发者工具
按 **F12** 或右键点击页面 → 选择"检查"

### 步骤 2：打开 Application 标签
```
┌─────────────────────────────────────┐
│ Elements  Console  Sources  Network │
│ Application ← 点击这里                │
└─────────────────────────────────────┘
```

### 步骤 3：清除站点数据
1. 在左侧找到 **Storage** 部分
2. 点击 **Clear site data** 按钮
3. 确认清除

```
Application
├── Storage  ← 展开
│   ├── Local Storage
│   ├── Session Storage
│   ├── IndexedDB
│   └── 🗑️ Clear site data  ← 点击这里
```

### 步骤 4：刷新页面
按 **F5** 刷新页面

---

## 🐛 故障排查

### 问题 1: 刷新后还是看不到

**解决方案**：
1. 完全关闭浏览器
2. 重新打开浏览器
3. 访问 http://localhost:5173
4. 按照上述步骤操作

---

### 问题 2: 下拉菜单只有"OpenAI"等默认配置

**可能原因**：代码没有重新编译

**解决方案**：
1. 在终端停止开发服务器（Ctrl + C）
2. 重新启动：
   ```bash
   npm run dev
   ```
3. 刷新浏览器页面

---

### 问题 3: 配置列表是空的

**解决方案**：检查控制台错误
1. 按 **F12** 打开开发者工具
2. 切换到 **Console** 标签
3. 查看是否有红色错误信息
4. 截图发给我，我会帮你解决

---

## 🔍 如何确认配置已加载

### 方法 1：查看浏览器控制台
1. 按 **F12** 打开开发者工具
2. 在 **Console** 标签中输入：
   ```javascript
   localStorage.getItem('gpt-image-playground-settings')
   ```
3. 按 Enter，查看输出
4. 搜索 `"video-ds-2.0"` 字样

### 方法 2：查看 Application Storage
1. 按 **F12** 打开开发者工具
2. 切换到 **Application** 标签
3. 左侧 **Local Storage** → 展开 → 点击你的域名
4. 找到 `gpt-image-playground-settings` 键
5. 查看值中是否包含 `video-ds-2.0`

## 💡 小贴士

如果你经常遇到缓存问题，可以：

1. **使用无痕模式测试**：
   - Chrome: Ctrl + Shift + N
   - Firefox: Ctrl + Shift + P
   - Safari: Cmd + Shift + N

2. **禁用缓存（开发模式）**：
   - F12 → Network 标签
   - 勾选 "Disable cache"
   - 保持开发者工具打开
